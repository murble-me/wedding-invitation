# Ксения & Максим — Свадебное приглашение

Персонализированный сайт-приглашение на свадьбу. Каждый гость получает уникальную ссылку с персональным обращением. Сайт включает обратный отсчёт, анимации, форму подтверждения присутствия, кнопку с музыкальным сопровождением и поддержку светлой/тёмной темы.

## Демо

[Открыть демо](https://wedding-sage-seven.vercel.app)

---

## Возможности

| Секция | Описание |
|--------|----------|
| **Титульная страница** | Полноэкранный баннер с именами пары, датой свадьбы и живым обратным отсчётом до события |
| **Приглашение** | Персональное обращение к гостю по имени; на демо-странице отображается в общем виде |
| **Фотографии** | Галерея фотографий пары с плавными переходами |
| **Детали события** | Время, место, адрес и кнопка открытия карты |
| **Программа дня** | Анимированная временная шкала всех событий свадебного дня |
| **Дресс-код** | Интерактивная цветовая палитра с рекомендациями по стилю |
| **RSVP** | Форма подтверждения присутствия с валидацией, сохранением в Firebase Firestore и fallback в localStorage |

**Дополнительно:**
- Тёмная тема по умолчанию; переключатель сохраняет выбор в `localStorage`
- Анимация лепестков и искр (CSS + SVG + scroll-driven)
- Scroll reveal: элементы появляются при прокрутке через `IntersectionObserver`
- Фоновая музыка с кнопкой включения/выключения
- Полная адаптивность: 375 / 768 / 1280 px

## Технологии

| Инструмент | Для чего используется |
|------------|-----------------------|
| **Nuxt 3** | SSR/SPA фреймворк; файловый роутинг, автоимпорт компонентов и composables, серверный рендеринг |
| **Vue 3** | Реактивный UI; `<script setup>` Composition API, `computed`, `ref`, `reactive` |
| **TypeScript** | Строгая типизация пропсов, composables и данных форм |
| **Tailwind CSS** | Утилитарные CSS-классы для раскладки и адаптивности |
| **CSS Custom Properties** | Система токенов для светлой/тёмной темы, единая шкала отступов (8px grid) |
| **Google Fonts** | Great Vibes (декоративные имена), Playfair Display (заголовки), Montserrat (интерфейс) |
| **Firebase Firestore** | Хранение ответов RSVP в реальном времени в облаке |
| **Vercel** | Хостинг и деплой; автоматическая сборка из GitHub |

### Система персонализации

Каждый гость получает уникальную ссылку вида `/имя_гостя`.  
При открытии страница ищет это имя в `guests.json` и отображает персональное обращение.  
Если имя не найдено — происходит редирект на главную страницу с общим видом.  
`guests.json` добавлен в `.gitignore` — приватные данные гостей не попадают в публичный репозиторий.

```
/mama_papa_k   →   "Дорогие Мама и Папа"
/lyiza         →   "Дорогая Луизочка"
/              →   демо-вид: «Дорогие Мама и Папа»
```
